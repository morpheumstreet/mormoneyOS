## Agent Loop (Detail)

The agent loop alternates between **waking**, **running**, and **sleeping**. The heartbeat daemon runs in the background regardless of agent state.

### State Machine

```
waking ──► running ──► sleeping ──► waking
   ▲          │            │          │
   │          │            │          │
   └──────────┴────────────┴──────────┘
```

- **Waking / Running:** The main loop calls `loop.RunOneTurn()` each iteration. When paused (dashboard), it sleeps `wakeCheck` (default 30s) and skips turns.
- **Sleeping:** The main loop does not run agent turns. It polls for wake conditions every `wakeCheck` (30s), then sleeps until the next check or `sleep_until` expiry.

### Per-Turn Flow (RunOneTurn)

Each turn follows the ReAct pattern:

1. **Survival tier** — Credits checked via `creditsFn`; `agent_state` and low-compute mode set from `conway.TierFromCreditsCents`. `ResourceConstraintMode` (forced_on/forced_off) overrides tier for model selection only.
2. **Input** — Claim inbox messages (social, etc.); if none, use wake prompt ("You are awake. What do you want to do next?" when turnCount > 0).
3. **Context** — System prompt (status, genesis, skills), recent turns, memory retrieval. `BuildMessagesSafe` enforces token cap (default 5500) and truncates history when over; see [token-caps-truncation.md](./token-caps-truncation.md).
4. **Inference** — Chat with tools; policy evaluates each tool call before execution.
5. **Outcome** — Persist turn and tool calls; return `TurnResult{State, WasIdle, Err}`.

### Sleep Triggers (3 Ways)

| Trigger | When | Source |
|---------|------|--------|
| **finishReason stop** | Model returns no tool calls and `finish_reason == "stop"` | `internal/agent/loop.go` (before tool loop) |
| **sleep tool** | Agent calls `sleep` and it succeeds | `internal/agent/loop.go` (after tool loop) |
| **Idle detection** | 3 consecutive turns with no mutating tools | `cmd/run.go` main loop; `loop.ShouldSleep(idleTurns)` |

Mutating tools are defined in `internal/tools/mutating.go`; read-only tools (e.g. `check_credits`, `system_synopsis`) do not reset idle count.

### Wake Triggers

| Trigger | Source |
|---------|--------|
| **Wake events** | Heartbeat tasks (e.g. `check_social_inbox`, low credits) call `InsertWakeEvent`; main loop checks `HasUnconsumedWakeEvents()` |
| **sleep_until expiry** | `sleep` tool or default 60s sets `sleep_until`; main loop wakes when time passes |

### Heartbeat During Sleep

The heartbeat daemon continues ticking while the agent sleeps. Tasks such as `check_social_inbox` (every 10s by default) poll channels; new messages are persisted to `inbox_messages` and a wake event is inserted. The main loop wakes within the next `wakeCheck` (30s), claims inbox messages, and the agent can reply via `send_message`.

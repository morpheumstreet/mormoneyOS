# Contributing

Thanks for contributing to MoneyClaw.

## Good First Contributions

- improve docs clarity
- fix typos or broken links
- add better examples
- improve public skill wording
- suggest safer onboarding copy

## Before Opening A PR

Please make sure:

- the change fits the public scope of this repo
- no secrets, internal-only notes, or production details are included
- docs and examples stay consistent
- wording does not encourage bypass, evasion, or unsafe payment behavior

## PR Guidance

- keep changes focused
- explain user impact clearly
- include screenshots for README or docs presentation changes when helpful
- keep public docs trust-first and user-facing

## Out Of Scope For This Repo

Do not submit:

- internal infrastructure changes
- production deployment instructions
- merchant-specific workaround playbooks
- anything that would leak private operations context
